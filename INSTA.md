# 인스타그램 경쟁 채널 외부 분석 보고서 구성안

> 📌 본 구성안은 **공개 데이터만으로** 작성 가능한 항목으로 구성되어 있습니다.
> 계정 소유자의 Instagram Insights 없이도 완전한 보고서를 만들 수 있습니다.

---

## 1. 분석 개요

- **분석 목적**: 경쟁 채널 벤치마킹 및 자사 콘텐츠 전략 수립 근거 확보
- **분석 기간**: YYYY.MM.DD ~ YYYY.MM.DD (최근 게시물 기준, 권장 3~6개월)
- **분석 대상 채널**: @계정명
- **데이터 수집 방법**: instagrapi (Python 오픈소스, 무료) — 상세 사양은 부록 D 참조
- **분석 엔진**: Claude API (텍스트 분석 + Vision 이미지 분석)
- **핵심 요약 (Executive Summary)**: 보고서 전체 내용을 3~5문장으로 압축

### 데이터 수집 범위

| 수집 항목 | 수집 가능 여부 | 비고 |
|-----------|:-----------:|------|
| 게시물 캡션 (텍스트) | ✅ | 전체 수집 가능 |
| 해시태그 | ✅ | 캡션에서 추출 |
| 좋아요 수 | ✅ | 게시물별 수집 |
| 댓글 수 | ✅ | 게시물별 수집 |
| 댓글 내용 | ✅ | 감성 분석에 활용 |
| 게시 날짜/시간 | ✅ | 빈도·타이밍 분석에 활용 |
| 게시물 이미지/영상 썸네일 | ✅ | 비주얼 톤 분석에 활용 |
| 팔로워 수 (현재) | ✅ | 수집 시점 스냅샷 |
| 게시물 유형 (이미지/캐러셀/릴스) | ✅ | 포맷 분류 가능 |
| 도달(Reach) / 노출(Impression) | 🔶 추정 | 팔로워 수 × 포맷별 도달 계수로 역산 |

| 팔로워 활동 시간대 | 🔶 추정 | 댓글 타임스탬프 분포로 역추정 |
| 프로필 방문 / 웹사이트 클릭 | ❌ | 신뢰도 낮아 제외 |

#### A-5. 팔로워 활동 시간대 추정 (추정 가능도: ★★★★☆)

댓글 타임스탬프를 수집하여 팔로워 활동 패턴을 역추정합니다.

**산출 방법**:
1. 전체 댓글의 작성 시간(KST 기준) 수집
2. 게시 직후 1~2시간 내 댓글에 가중치 × 2.0 부여 (초기 반응 = 활성 유저)
3. 시간대별(1시간 단위) 댓글 분포 히트맵 생성
4. 요일 × 시간대 매트릭스로 활동 패턴 도출

**방법론 근거**: Instagram 알고리즘이 초기 인게이지먼트(게시 후 30분 내 반응)를 우선시하며, 이 시간대에 반응한 사용자가 실제 활성 팔로워일 가능성이 높음 (Adam Mosseri 2025.01 공식 발언, Sprout Social 2025 Content Benchmarks)

**한계**: 댓글을 남기지 않는 조용한 팔로워(lurker)의 활동 패턴은 반영되지 않음

### B. 업종 프리셋 시스템 (콘텐츠 카테고리 + 경쟁 채널)

> 본 분석 도구는 **업종 프리셋**을 통해 다양한 업종의 채널 분석에 대응합니다.
> `--industry` 플래그로 프리셋을 선택하면 해당 업종에 맞는 카테고리 체계와 추정 보정 계수가 자동 적용됩니다.

#### B-1. 프리셋 구조

각 프리셋은 `presets/{업종}.yaml` 파일로 관리되며, 다음 3가지를 포함합니다:

| 항목 | 내용 | 역할 |
|------|------|------|
| **카테고리 정의** | 코드 10개 + 정의 + 분류 키워드 | Claude AI가 캡션을 분류할 때 참조 |
| **추정 보정 계수** | 카테고리별 저장/공유 계수 보정값 | 업종 특성 반영 (교육형 콘텐츠 저장↑ 등) |
| **경쟁 채널 리스트** | 벤치마킹 대상 채널 목록 (선택사항) | 다중 채널 분석 시 활용 |

**프리셋 선택 방식**:

```bash
python main.py @omuk_food --industry food       # 저장된 프리셋 사용
python main.py @channel --industry auto          # AI가 자동 생성 (아래 B-5 참조)
python main.py @channel --industry custom:my.yaml  # 사용자 정의 프리셋
```

#### B-5. 자동 프리셋 생성 (`--industry auto`)

`auto` 모드에서는 Claude AI가 수집된 게시물을 먼저 분석하여 카테고리 체계를 자동 생성합니다.

**처리 흐름**:

```
[1] 게시물 최근 50개 캡션 샘플링
  → [2] Claude API (Sonnet)에 전달:
       "이 채널의 게시물 캡션을 분석하여
        10개 이내의 콘텐츠 카테고리를 생성하라.
        각 카테고리에 코드, 이름, 정의, 분류 키워드,
        저장 보정 계수, 공유 보정 계수를 포함하라."
  → [3] 생성된 카테고리 체계를 presets/auto_{채널명}.yaml로 저장
  → [4] 해당 프리셋으로 나머지 파이프라인 진행
```

**추가 비용**: Sonnet 1회 호출 (~$0.15), 전체 파이프라인 비용에 합산

**auto 모드가 유용한 경우**:
- 기존 프리셋에 해당하지 않는 업종 (부동산, 교육, 여행, 피트니스 등)
- 업종 경계가 모호한 라이프스타일 채널
- 처음 보는 업종의 채널을 빠르게 분석할 때

| 수집 데이터 | instagrapi 메서드/속성 | 보고서 적용 섹션 |

| 댓글 수 | `Media.comment_count` | §4, §6 인기 게시물 |
| 댓글 텍스트 + 작성자 + 시간 | `Client.media_comments()` → `.text`, `.user.username`, `.created_at_utc` | §3 오디언스 추정, §6 감성 분석 |
| 게시 날짜/시간 | `Media.taken_at` | §7 게시 빈도 & 타이밍 |
| 게시물 유형 | `Media.media_type` (1=Image, 2=Video, 8=Sidecar) | §4 포맷별 분석 |
| 이미지 다운로드 | `Client.photo_download()` / `Client.album_download()` | §5 비주얼 톤 분석 |
| 영상 썸네일 URL | `Media.thumbnail_url` | §5 비주얼 톤 분석 |
| 캐러셀 슬라이드 수 | `len(Media.resources)` | §4 포맷별 분석 |
| 팔로워 수 | `UserInfo.follower_count` | 추정치 산출 기준 |
| 팔로잉 수 | `UserInfo.following_count` | §2 채널 프로필 |
| 게시물 총 수 | `UserInfo.media_count` | §2 채널 프로필 |
| 프로필 사진 | `UserInfo.profile_pic_url` | §2 채널 프로필 |
| 계정 인증 배지 | `UserInfo.is_verified` | §2 채널 프로필 |
| 비즈니스 카테고리 | `UserInfo.category_name` | §2 채널 프로필 |

**실행 예시 (Python)**:

```python
from instagrapi import Client
import time

cl = Client()
cl.login("username", "password")
user_info = cl.user_info_by_username("omuk_food")

# 프로필 정보
print(f"팔로워: {user_info.follower_count}")
print(f"Bio: {user_info.biography}")

# 최근 게시물 200개 수집
medias = cl.user_medias(user_info.pk, amount=200)
posts = []
for media in medias:
    posts.append({
        "date": media.taken_at,
        "caption": media.caption_text,
        "likes": media.like_count,
        "comments": media.comment_count,
        "type": media.media_type,  # 1=Image, 2=Video, 8=Sidecar
        "url": str(media.thumbnail_url)
    })
    time.sleep(2)  # IP 차단 방지 딜레이
```

| 1회 수집 게시물 수 | 최대 200개 | 과도한 요청 방지 |
| 로그인 여부 | 로그인 권장 | 비로그인 시 수집 제한 더 엄격 |
| 세션 재사용 | `Client.load_settings()` / `Client.login()` | 반복 로그인 방지 |
| 프록시 | 대량 수집 시 권장 | 동일 IP 차단 우회 |

#### D-4. AI 분석 — Claude API 모델 배정 전략

**사용 모델 및 가격**:

| 모델 | API model ID | Input (1M 토큰) | Output (1M 토큰) | 역할 |
|------|-------------|:-:|:-:|------|
| **Haiku 4.5** | `claude-haiku-4-5-20251001` | $1 | $5 | 분류·추출·감성 분석 등 단순 작업 |
| **Sonnet 4.6** | `claude-sonnet-4-6` | $3 | $15 | 다단계 추론, 종합 분석, Vision 이미지 분석 |

> 설치: `pip install anthropic`

**모델 배정 기준**:
- 패턴 매칭·분류·추출 작업 → **Haiku** (Sonnet 대비 2~5% 이내 정확도, 73% 저렴)
- 다단계 추론·뉘앙스 해석·창의적 종합 → **Sonnet** (추론 깊이 필요)
- Vision (이미지 분석) → **Sonnet** (Haiku도 Vision 지원하나, 복합 미적 판단은 Sonnet 권장)

**보고서 섹션별 모델 배정표**:

| 추정 전체 인게이지먼트율 | (좋아요 + 댓글 + 추정 저장 + 추정 공유) / 추정 조회 × 100 | 복합 산출 |

### F. 데이터 한계 및 면책 사항

- 본 보고서는 공개 데이터 + 업계 벤치마크 기반 추정치로 작성되었습니다.
- 🔶 추정 표기된 지표의 계수는 글로벌 평균 데이터 기반이며, 한국 시장/푸드 미디어 특화 계수는 아닙니다.
- 저장/공유 계수의 출처 데이터는 주로 글로벌 브랜드 대상이며, 한국 푸드 미디어 계정에는 편차가 있을 수 있습니다.
- Socialinsider 조회 수는 "Views(조회)"이며, "Reach(순 도달)"과 다릅니다. 순 도달은 조회 수보다 낮습니다.
- 공유 수는 2024~2025년 사이 급증 추세(Dash Social 기준 +86%)이므로, 분석 시점에 따라 계수 재보정이 필요합니다.
- Instagram 알고리즘은 지속적으로 변하며, 2025년 기준 데이터가 분석 시점에 유효하지 않을 수 있습니다.
- 프로필 방문 / 웹사이트 클릭은 신뢰할 수 있는 추정 방법이 없어 제외하였습니다.

### G. 용어 정의

| 용어 | 정의 |
|------|------|
| 추정 인게이지먼트율 | (좋아요 + 댓글) / 팔로워 수 × 100 |
| 추정 전체 인게이지먼트율 | (좋아요 + 댓글 + 추정 저장 + 추정 공유) / 추정 도달 × 100 |
| 저장 계수 | 좋아요 대비 저장 수 추정 비율 (카테고리별 상이) |
| 공유 계수 | 좋아요 대비 공유 수 추정 비율 (카테고리별 상이) |
| 도달 계수 | 팔로워 대비 도달 수 추정 비율 (포맷별 상이) |
| 캐러셀 | 여러 장의 이미지/영상을 한 게시물에 담은 슬라이드형 콘텐츠 |
| 릴스 | Instagram의 숏폼 동영상 포맷 (최대 90초) |

| UGC | User Generated Content. 사용자가 자발적으로 생성한 콘텐츠 |
| 니치 해시태그 | 게시물 수 1만 개 이하의 소규모 해시태그 |
